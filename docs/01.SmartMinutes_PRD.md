# 產品需求文件 (PRD)：AI 自動化會議紀錄助手 (SmartMinutes)

## 1. 文件資訊

- **專案名稱**：SmartMinutes AI 自動化會議紀錄程式
- **版本**：v1.0 (MVP)
- **狀態**：草案 (Draft)
- **日期**：2026-04-25

---

## 2. 專案背景與目標

### 2.1 背景

現代工作環境會議頻繁，手動整理會議紀錄耗時且容易遺漏重點。市面上雖有轉錄工具，但缺乏針對台灣工作場景優化的「語音轉文字 + 智慧摘要 + 字幕生成」的一站式解決方案。

### 2.2 目標

開發一個自動化工具，使用者上傳會議音檔後，系統能自動完成：

1. **精準轉錄**：將語音轉為逐字稿。
2. **語者辨識**：區分會議中不同的說話者。
3. **智慧摘要**：利用 LLM 產出結構化會議結論。
4. **多格式輸出**：提供逐字稿、摘要報告及 SRT 字幕檔。

---

## 3. 用戶需求分析 (User Stories)

1. **專案經理 (PM)**：我希望上傳週會錄音後，AI 能直接列出 Action Items，讓我不用重聽音檔。
2. **訪談研究員**：我希望逐字稿能標註 Speaker A/B，並支援點擊文字即播放該段音頻，方便校對。
3. **影音剪輯師**：我希望系統能生成帶有精準時間軸的 SRT 字幕檔，直接匯入剪輯軟體。

---

### 4.2 摘要模板系統

系統需提供預設 Prompt 模板：

- **一般週會**：側重決議與進度。
- **訪談/面試**：側重 Q&A 對話結構。
- **技術腦力激盪**：側重解決方案與技術細節。

---

## 5. 技術架構 (Technical Architecture)

### 5.1 推薦技術棧

- **Frontend**: Next.js 14, Tailwind CSS, Lucide React (UI Icon).
- **Backend**: Python FastAPI (異步處理大檔案轉錄).
- **Task Queue**: Celery + Redis (處理長時間的音檔運算任務).
- **Audio Engine**: FFmpeg (格式轉碼).
- **AI Models**:
  - **ASR**: OpenAI Whisper (Large-v3).
  - **Diarization**: Pyannote-audio.
  - **LLM**: GPT-4o 或 Claude 3.5 Sonnet (用於摘要).
- **Storage**: AWS S3 或 Google Cloud Storage (存放音檔).

---

## 6. 介面設計構思 (UI/UX)

1.  **上傳區**：簡潔的拖放介面，顯示上傳進度條。
2.  **處理中**：顯示目前的步驟（如：轉錄中、語者分析中、AI 摘要中）。
3.  **工作台 (Workspace)**：
    - **左側欄**：音頻播放控制器（1.5x/2x 倍速、倒退 5 秒）。
    - **中間欄**：逐字稿內容，點擊句子音軌會自動跳轉。
    - **右側欄**：AI 生成的摘要區域，點擊「重新生成」可切換模板。

---

## 7. 資料庫設計 (Database Schema)

### 7.1 會議紀錄表 (Meetings)

- `id`: UUID
- `user_id`: ForeignKey
- `file_url`: String (S3 連結)
- `status`: Enum (Processing, Completed, Failed)
- `transcript`: Text (原始逐字稿 JSON，含時間戳)
- `summary`: Text (AI 生成摘要)
- `created_at`: DateTime

---

## 8. 隱私與安全

- **傳輸安全**：所有 API 調用必須使用 HTTPS 加密。
- **隱私保護**：音檔在轉錄完成後的指定時間（如 24 小時）自動從服務器刪除。
- **合規性**：提供「隱私遮罩」功能，摘要時自動去識別化敏感個資。

---

## 9. 階段規劃 (Roadmap)

- **Phase 1 (MVP)**：完成上傳、Whisper 轉錄、簡單摘要、TXT 匯出。
- **Phase 2 (優化)**：加入語者辨識 (Diarization)、SRT 字幕生成、線上編輯介面。
- **Phase 3 (擴張)**：串接 Google Drive/OneDrive，支援即時翻譯功能。
