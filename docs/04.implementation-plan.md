## 4. 技術風險與緩解策略

| 風險 | 影響 | 緩解策略 |
|:---|:---|:---|
| 模型推論需佔用資源 | 伺服器無 GPU 時轉錄較慢 | 採用 Faster-Whisper 降低資源消耗，或使用較小模型 |
| 長音檔記憶體溢出 | 處理失敗 | 音檔分段處理（每段 30 分鐘），分段合併結果 |
| LLM Token 長度限制 | 長逐字稿無法一次送入 | 分段摘要 + 最終彙整（Map-Reduce 策略） |
| Pyannote 授權限制 | 模型下載需 HuggingFace Token | 文件中明確說明申請流程 |
| 中文辨識準確度 | 專有名詞錯誤率高 | 提供線上校對編輯器讓使用者手動修正 |

## 5. MVP 範圍界定

### ✅ MVP 包含 (P0)

- 音檔上傳（MP3/WAV/M4A/MP4）
- Faster-Whisper 語音轉文字（繁中+中英夾雜）
- AI 摘要生成（3 種預設模板）
- 逐字稿 / 摘要線上檢視
- 匯出下載：TXT、Markdown、SRT、PDF、Word
- 處理進度即時顯示

### ⏳ Phase 2 再做 (P1)

- 語者辨識 (Speaker Diarization)
- 線上校對編輯器（點擊跳轉音軌）
- VTT 字幕格式
- （MVP 為單機版，使用者帳號系統延後至團隊版規劃）

### 🔮 Phase 3 未來 (P2)

- Google Drive / OneDrive 整合
- 即時翻譯
- 自訂 Prompt 模板
- 會議歷史管理與搜尋
