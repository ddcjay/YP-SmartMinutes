## 4. 核心處理流程

```
使用者上傳音檔
      │
      ▼
  ① 檔案驗證與儲存
      │ (格式/大小檢查, 儲存至暫存區)
      ▼
  ② FFmpeg 音訊預處理
      │ (MP4/M4A → WAV 16kHz mono)
      ▼
  ③ Whisper ASR 轉錄
      │ (產出含時間戳的逐字稿 JSON)
      ▼
  ④ Pyannote 語者辨識 (P1)
      │ (標註 Speaker 0, 1, 2...)
      ▼
  ⑤ 逐字稿合併與對齊
      │ (將語者標籤對應至轉錄文字段落)
      ▼
  ⑥ LLM 摘要生成
      │ (產出「摘要總覽+待辦項目+重點記錄+建議行動」格式)
      ▼
  ⑦ SRT/VTT 字幕生成
      │ (根據時間戳格式化)
      ▼
  ⑧ 結果儲存 & 通知前端
      │ (WebSocket 推播完成狀態)
      ▼
  使用者檢視/下載成果
```

## 5. API 設計概要

### 5.1 核心端點

| 方法 | 路徑 | 說明 |
|:---|:---|:---|
| POST | `/api/meetings/upload` | 上傳音檔，建立會議任務 |
| GET | `/api/meetings/{id}` | 取得會議詳情與處理狀態 |
| GET | `/api/meetings/{id}/transcript` | 取得逐字稿 |
| GET | `/api/meetings/{id}/summary` | 取得 AI 摘要 |
| POST | `/api/meetings/{id}/summary/regenerate` | 切換模板重新生成摘要 |
| PUT | `/api/meetings/{id}/transcript` | 更新修改後的逐字稿 |
| GET | `/api/meetings/{id}/export/{format}` | 匯出檔案 (srt/vtt/txt/pdf/docx/md) |
| WS | `/ws/meetings/{id}/progress` | WebSocket 即時進度推播 |

### 5.2 統一回應格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## 7. 安全設計

| 面向 | 措施 |
|:---|:---|
| 傳輸安全 | 全站 HTTPS，API 強制 TLS |
| 檔案安全 | 上傳檔案類型白名單驗證、檔案大小限制、防毒掃描 |
| API 安全 | JWT 認證、Rate Limiting、CORS 白名單 |
| 資料隱私 | 音檔定時清理（預設 24h）、摘要去識別化選項 |
| 密鑰管理 | 所有 API Key 透過環境變數注入，禁止寫死 |
