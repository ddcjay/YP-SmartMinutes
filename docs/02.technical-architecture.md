## 3. 技術棧詳細選型

### 3.1 前端 (Frontend)

| 技術 | 版本 | 用途 |
|:---|:---|:---|
| Next.js | 14 (App Router) | 前端框架，支援 SSR/SSG |
| TypeScript | 5.x | 型別安全 |
| Tailwind CSS | 4.x | 樣式系統 |
| Lucide React | latest | UI Icon 圖示庫 |
| shadcn/ui | latest | UI 元件庫 |
| EventSource API | 瀏覽器原生 | SSE 即時進度接收（零額外套件） |
| Wavesurfer.js | 7.x | 音頻波形視覺化與播放控制 |

### 3.2 後端 (Backend)

| 技術 | 版本 | 用途 |
|:---|:---|:---|
| Python | 3.11+ | 主要開發語言 |
| FastAPI | 0.110+ | Web 框架（異步支援） |
| SQLAlchemy | 2.x | ORM（支援 SQLite↔PostgreSQL 無痛遷移） |
| ThreadPoolExecutor | Python 內建 | 背景任務處理（取代 Celery） |

### 3.3 AI / 音訊處理

| 技術 | 用途 |
|:---|:---|
| FFmpeg | 音訊格式轉碼（MP4/M4A → WAV 16kHz mono） |
| Faster-Whisper | ASR 語音轉文字（本地免費部署、速度快） |
| Google Gemini API (Free Tier) | LLM 摘要生成（免費額度，個人使用足夠） |

### 3.4 基礎設施

| 技術 | 用途 |
|:---|:---|
| SQLite | 內嵌式資料庫（零設定、隨專案備份） |
| 本地檔案系統 | 音檔儲存 |

> **NOTE**：未來升級團隊版時，僅需更換 SQLAlchemy 的 connection string 即可遷移至 PostgreSQL。

## 4. 核心處理流程

```
使用者上傳音檔
      │
      ▼
  ① 檔案驗證與儲存
      │ (格式/大小檢查, 儲存至 uploads/)
      ▼
  ② FFmpeg 音訊預處理
      │ (MP4/M4A → WAV 16kHz mono)
      ▼
  ③ Faster-Whisper ASR 轉錄
      │ (產出含時間戳的逐字稿，寫入 SQLite)
      ▼
  ④ Gemini API 摘要生成
      │ (產出「摘要總覽+待辦項目+重點記錄+建議行動」)
      ▼
  ⑤ 結果儲存 & SSE 通知前端
      │ (前端透過 EventSource 接收完成通知)
      ▼
  使用者檢視/下載成果
```

## 5. API 設計概要

### 5.1 核心端點

| 方法 | 路徑 | 說明 |
|:---|:---|:---|
| POST | `/api/meetings/upload` | 上傳音檔，建立會議任務 |
| GET | `/api/meetings/{id}` | 取得會議詳情（含逐字稿與摘要） |
| GET | `/api/meetings/{id}/progress` | SSE 即時進度推播 |
| POST | `/api/meetings/{id}/summary/regenerate` | 切換模板重新生成摘要 |
| GET | `/api/meetings/{id}/export/{format}` | 匯出檔案 (srt/txt/summary/md) |
| GET | `/health` | 健康檢查 |

### 5.2 統一回應格式

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## 6. 資料庫設計 (SQLite)

### 6.1 資料表結構

```
┌──────────────────┐       ┌────────────────┐
│    meetings      │       │   transcripts  │
├──────────────────┤       ├────────────────┤
│ id (PK, UUID)    │──┐    │ id (PK, auto)  │
│ title            │  ├──▶ │ meeting_id(FK) │
│ file_path        │  │    │ speaker_label  │
│ file_size        │  │    │ start_time     │
│ duration         │  │    │ end_time        │
│ status           │  │    │ content         │
│ progress         │  │    │ is_edited       │
│ progress_message │  │    │ created_at      │
│ template_type    │  │    └────────────────┘
│ created_at       │  │
│ updated_at       │  │    ┌────────────────┐
└──────────────────┘  │    │   summaries    │
                      │    ├────────────────┤
                      └──▶ │ id (PK, auto)  │
                           │ meeting_id(FK) │
                           │ template_type  │
                           │ content (JSON) │
                           │ created_at     │
                           └────────────────┘
```

### 6.2 meetings.status 狀態機

```
uploading → processing → transcribing → summarizing → completed
                                                  ↘ failed
```

## 7. 安全設計

| 面向 | 措施 |
|:---|:---|
| 檔案安全 | 上傳檔案類型白名單驗證（MP3/WAV/M4A/MP4）、大小限制 500MB |
| API 安全 | CORS 白名單、全域錯誤攔截（不洩露內部資訊） |
| 資料隱私 | 音檔定時清理（預設 24h）、所有資料儲存於本機 |
| 密鑰管理 | Gemini API Key 透過環境變數注入，禁止寫死 |
