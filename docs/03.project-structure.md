## 2. 關鍵設計說明

### 2.1 前端分層

- **app/** → 頁面路由，僅處理佈局與頁面組裝
- **components/** → 純 UI 元件，按功能模組分資料夾
- **hooks/** → 業務邏輯封裝，將狀態管理從元件中抽離
- **lib/** → 共用工具與 API 呼叫封裝

### 2.2 後端分層

- **api/** → HTTP 路由，僅負責請求驗證與回應格式
- **services/** → 核心業務邏輯（單一職責原則）
- **models/** → 資料庫 ORM 模型定義
- **schemas/** → 請求/回應的資料格式驗證
- **tasks/** → Celery 非同步任務定義

### 2.3 Prompt 模板管理

將 LLM Prompt 以獨立 `.txt` 檔管理，便於非工程人員調整用語與格式，無需修改程式碼。
