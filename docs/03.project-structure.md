## 2. 關鍵設計說明

### 2.1 前端分層

- **app/** → 頁面路由，僅處理佈局與頁面組裝
- **components/** → 純 UI 元件，按功能模組分資料夾
- **hooks/** → 業務邏輯封裝（SSE 訂閱、上傳、音頻播放）
- **lib/** → 共用工具與 API 呼叫封裝

### 2.2 後端分層

- **api/** → HTTP 路由，僅負責請求驗證與回應格式
- **services/** → 核心業務邏輯（單一職責原則）
- **models/** → SQLAlchemy ORM 模型定義
- **schemas/** → Pydantic 請求/回應資料格式驗證
- **tasks/** → 背景處理（ThreadPoolExecutor，取代 Celery）
- **db/** → SQLite 連線與 Session 管理

### 2.3 精簡架構特點

| 原設計 | 方案 B | 效益 |
|:---|:---|:---|
| Celery + Redis | ThreadPoolExecutor | 少啟動 2 個服務 |
| PostgreSQL | SQLite | 零設定、零 Docker |
| WebSocket (Socket.IO) | SSE (EventSource) | 零額外套件 |
| 4 個服務啟動 | **1 個服務啟動** | 大幅降低部署複雜度 |
